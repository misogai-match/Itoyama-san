📋 週次更新手順

まっちがClaude Codeに依頼するだけ

「ダッシュボード更新して」で以下が自動実行される：

  • 最新の入出金サマリExcelをGドライブから読み込み
  • Python実行 → 日次データ(daily_data.json) + 報告データ(report_data.json) 生成
  • cashflow_dashboard.html のインラインデータを更新
  • ヘッダーの「データ: X/XX時点 | 更新: X/XX」を更新
  • git commit & push → GitHub Pages に反映（数分で反映）
  • URLを返す → まっちがブラウザで確認
💡 所要時間: 約3〜5分。まっちの作業は「依頼」と「確認」だけ。
📋 月替わり対応手順

月初にやること（4月→5月の例）

  • 城さん・心ちゃんがExcelに新月シート（「2026年04月予測」）を追加
  • Python (extract_daily_data.py) に新月シートを追加対応
  • apr_forecast のデータを新月の予測値に更新
    • INTER: 直近2ヶ月平均で更新
    • HUBRIDE売上: KINTONE登録の入金予定額を反映
    • HUBRIDE仕入: 鈴木さんと相談して台数確定
    • HUBRIDE経費: 直近3ヶ月平均で更新
  • 残高チェーン検証（前月末残高 → 当月末残高の整合性）
  • 仕入シミュレーターの台単価を見直し（5ヶ月平均を更新）
  • デプロイ・動作確認
⚠ 月替わりは完全自動化が難しい（予測値の判断が必要）。まっち＋Cで対応。所要時間: 約30分。
